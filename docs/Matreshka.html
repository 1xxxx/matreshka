<static> elementProcessors :function

Matreshka.elementProcessors is the array of functions that compare given element with rules. This using for defining elements behavior in Matreshka#bindElement method.
Type:
  • function
Source:
  • matreshka.js, line 1044
Examples

HTML5 input type number

//shift means that we're adding new elementProcessor to the beginning of MK.elementProcessors list
MK.elementProcessors.shift( function( element ) {
	if( element.tagName === 'input' && element.type === 'number' ) return {
		on: 'mouseup',
		getValue: function() {
			return this.value;
		},
		setValue: function( v ) {
			this.value = v;
		}
	};
});
this.bindElement( 'myKey', '.my-input-type-number' );

Custom checkbox

MK.elementProcessors.shift( function( element ) {
	if( $( element ).hasClass( 'custom-checkbox' ) ) return {
		on: 'click',
		getValue: function() {
			return $( this ).hasClass( 'checked' );
		},
		setValue: function( v ) {
			$( this ).toggleClass( 'checked', !!v );
		}
	};
});
this.bindElement( 'myKey', '.custom-checkbox' );

<static> htmlp :elementOptions

innerHTML element processor

By default if you pass html element to Matreshka#bindElement that doesn't match any elementProcessor, the binding does nothing. But sometimes you want to change innerHTML without having possibility to retrieve value from the element. Matreshka.htmlp is created as simple elementOptions object for this case for redusing your code.
Type:
  • elementOptions

Example

Usage

this.bindElement( 'myKey', '.my-element', MK.htmlp );
// same as
this.bindElement( 'myKey', '.my-element', { // no "getValue" and no "on" property

		this.innerHTML = v;
	}

bindElement(key, el, elOpts, evtOpts) → {mk}

Creates event bridge between DOM node and Matreshka instance property

This powerful function binds element to instance property with given options that say when and how to extract element's value, how to set element's value when property is changed.
Parameters:
Name Type Argument Description
key string | mk The key (or keys separated by space) that has to be binded to given element(s)
el Array.<Node> | NodeList | Node | jQuery | string The element (DOM Node or DOM NodeList or array of nodes or jQuery instance or css selector) that has to be binded to given key(s)
elOpts elementOptions <optional>
 Element options object that contains: setValue (how to set value for an element), getValue (how to extract value from element), on (when we have to extract value from element and put it to the property)
evtOpts eventOptions <optional>
 If you want to set "silent" flag or pass some options to "bind" event handler
Source:
  • matreshka.js, line 241
Fires:
  • event:bind
Returns:
self
Type
mk
Examples

Basic usage 1

this.bindElement( 'myKey', 'input[type="checkbox"]', {
	on: 'click',
	getValue: function() {
		return this.checked; // "this" is checkbox element  
	},
	setValue: function( v ) {
		this.checked = !!v;
	}
});
// now when element is binded you can change myKey and look at changes in UI
this.myKey = true; // makes checkbox checked
this.myKey = false; // makes checkbox unchecked

Basic usage 2. By Default Matreshka.elementProcessors array contains elementOptions for few dom elements (input[type="text"], input[type="radio"], input[type="checkbox"], select, textarea...). So you don't need to pass eventOptions for these elements

this.bindElement( 'myKey', '.checkbox' );

Custom checkbox 1. This example Shows how to create your own custom checkbox that has "checked" class if checked.

this.bindElement( 'myKey', '.custom-checkbox', {
	on: 'click',
	getValue: function() {
		// at this case we use jQuery
		return $( this ).hasClass( 'checked' );
	},
	setValue: function( v ) {
		$( this ).toggleClass( 'checked', !!v );
	}
});

Custom checkbox 2. In this example we'll do the same as in previous example but using predefined elementOptions via Matreshka.elementProcessors.

//shift means that we're adding new elementProcessor to the beginning of MK.elementProcessors list
MK.elementProcessors.shift( function( element ) {
	if( $( element ).hasClass( 'custom-checkbox' ) ) return {
		on: 'click',
		getValue: function() {
			return $( this ).hasClass( 'checked' );
		},
		setValue: function( v ) {
			$( this ).toggleClass( 'checked', !!v );
		}
	};
	})
this.bindElement( 'myKey', '.custom-checkbox' );

Using "bind" event

this.on( 'bind:myKey', function() { alert( 'ok!' ); });
this.bindElement( 'myKey', '.custom-checkbox' ); // alerts "ok!"

Using "bind" event options

this.on( 'bind:myKey', function() { alert( 'ok!' ); });
this.bindElement( 'myKey', '.custom-checkbox', {}, { silent: true } ); // no alert

Extending default elementOptions. For example we're working with input[type="text"]. By default "on" property contains "keydown". But we want to use "blur" event for changing myKey

this.bindElement( 'myKey', '.custom-checkbox', { on: "blur" });

Binding self to the element. If you want to use context (sandbox) for binding elements contained in single element, you can pass this to the method

// you can use this.bindElement( '__this__', '.app' ); instead
this.bindElement( this, '.app' );
// this.$( '.my-element' ) takes element(s) from .app
this.bindElement( 'myKey', this.$( '.my-element' ) );
